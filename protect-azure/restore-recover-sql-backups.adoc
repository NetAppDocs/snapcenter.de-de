= Stellen Sie SQL Server Datenbanken wieder her
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Mit SnapCenter können Sie SQL Server-Datenbanken wiederherstellen.  Die Wiederherstellung einer Datenbank ist ein mehrphasiger Prozess, bei dem alle Daten und Protokollseiten aus einer bestimmten SQL-Server-Sicherung in eine bestimmte Datenbank kopiert werden.

.Bevor Sie beginnen
* Konfigurieren Sie die Zielinstanz mit einem Active Directory-Benutzer, der zur SMB Active Directory-Domäne gehört und über die Berechtigung verfügt, die Dateiberechtigungen entsprechend festzulegen.
* Konfigurieren Sie die Anmeldeinformationen auf SnapCenter Instanzebene.
* Wenn sich die SQL-Datenbank auf einer CIFS-Freigabe befindet, müssen das RunAS-Konto und das SQL-Dienstkonto (Domänenkonto) des Plug-in-Hosts Teil der Konfiguration sein. `BUILTIN\Administrators` Gruppe des CIFS-Servers in ONTAP.
* SMB-Konfigurationen unterstützen keine SQL-Authentifizierung für die Zielinstanz.  Konfigurieren Sie die Zielinstanz in SnapCenter mit einem Active Directory-Benutzer, der über die erforderlichen Berechtigungen verfügt.
* Wenn es sich bei dem SnapCenter Plug-in-Dienstkonto nicht um einen Active Directory-Benutzer handelt, stellen Sie sicher, dass beim Wiederherstellen auf einem alternativen Host ein Benutzer mit vollständiger Kontrolle über die Quellvolumes verfügbar ist.


.Schritte
. Klicken Sie im linken Navigationsbereich auf *Ressourcen* und wählen Sie dann das entsprechende Plug-in aus der Liste aus.
. Wählen Sie auf der Seite Ressourcen entweder *Datenbank* oder *Ressourcengruppe* aus der Liste Ansicht aus.
. Wählen Sie die Datenbank oder die Ressourcengruppe aus der Liste aus.
. Wählen Sie in der Ansicht Manage Copies die Option *Backups* aus dem Speichersystem aus.
. Wählen Sie das Backup aus der Tabelle aus, und klicken Sie dann auf das image:../media/restore_icon.gif["Symbol Wiederherstellen"] Symbol.
. Wählen Sie auf der Seite „Bereich wiederherstellen“ eine der folgenden Optionen aus:
+
.. Wählen Sie *Datenbank auf demselben Host wiederherstellen, auf dem die Sicherung erstellt wurde*, um die Datenbank auf demselben SQL-Server wiederherzustellen.
.. Wählen Sie *Datenbank auf einem alternativen Host wiederherstellen*, wenn Sie die Datenbank auf einem anderen SQL-Server auf demselben oder einem anderen Host wiederherstellen möchten, auf dem die Sicherungen erstellt werden.


. Wählen Sie auf der Seite „Recovery Scope“ eine der folgenden Optionen aus:
+
.. Wählen Sie *Keine* aus, wenn Sie nur das vollständige Backup ohne Protokolle wiederherstellen müssen.
.. Wählen Sie *Alle Protokollsicherungen* Up-to-the-minute Backup Restore Operation, um alle verfügbaren Protokollsicherungen nach dem vollständigen Backup wiederherzustellen.
.. Wählen Sie *nach Log-Backups*, um einen Point-in-Time-Wiederherstellungsvorgang durchzuführen, der die Datenbank basierend auf Backup-Protokollen bis zum ausgewählten Datum wiederherstellt.
.. Wählen Sie *nach einem bestimmten Datum bis*, um Datum und Uhrzeit anzugeben, nach denen Transaktionsprotokolle nicht auf die wiederhergestellte Datenbank angewendet werden.
.. Wenn Sie *Alle Log-Backups*, *durch Log-Backups* oder *nach einem bestimmten Datum bis* ausgewählt haben und sich die Protokolle an einem benutzerdefinierten Speicherort befinden, wählen Sie *Benutzerdefiniertes Log-Verzeichnis verwenden* und geben Sie dann den Speicherort an.


. Geben Sie auf der Seite Pre-Ops und Post Ops die erforderlichen Details an.
. Wählen Sie auf der Benachrichtigungsseite aus der Dropdown-Liste *E-Mail-Präferenz* die Szenarien aus, in denen Sie die E-Mails versenden möchten.
. Überprüfen Sie die Zusammenfassung und klicken Sie dann auf *Fertig stellen*.
. Überwachen Sie den Wiederherstellungsprozess mithilfe der Seite *Monitor* > *Jobs*.
