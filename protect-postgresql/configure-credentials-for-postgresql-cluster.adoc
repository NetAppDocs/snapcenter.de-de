= Konfigurieren Sie die Anmeldeinformationen für den PostgreSQL-Cluster
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie müssen die Clusterbenutzer-Anmeldeinformationen für den PostgreSQL-Cluster konfigurieren, um Datensicherungsoperationen durchzuführen.

.Bevor Sie beginnen
* Sind für den PostgreSQL-Cluster keine gültigen Anmeldeinformationen konfiguriert, wird ein rotes Vorhängeschloss-Symbol nur für die automatisch erkannten Ressourcen angezeigt. Wenn bei einer späteren Erkennung festgestellt wird, dass die konfigurierten Anmeldeinformationen falsch sind oder keinen Zugriff auf den Cluster selbst ermöglichen, wird das rote Vorhängeschloss-Symbol erneut angezeigt.
* Sie müssen gültige PostgreSQL-Cluster-Anmeldeinformationen konfigurieren, um die Ressource schützen oder sie einer Ressourcengruppe hinzufügen zu können, um Datensicherungsoperationen durchzuführen.
* Der in den Anmeldeinformationen konfigurierte PostgreSQL-Benutzer muss über ausreichende Berechtigungen für den Zugriff auf den Cluster sowie für Sicherungs- und Wiederherstellungsvorgänge verfügen. Mindestens sollte der Benutzer die Rolle *SUPERUSER* oder gleichwertige Berechtigungen (wie *pg_read_all_data*, *pg_write_all_data* und *replication*) haben.
* SnapCenter verwendet die konfigurierten Anmeldeinformationen, um über den *psql*- oder *libpq*-Client eine Verbindung zur PostgreSQL-Instanz herzustellen. Stellen Sie sicher, dass die PostgreSQL-Client-Tools auf dem Plug-in-Host installiert sind.



NOTE: Bevor Sie den Cluster konfigurieren, müssen Sie zuerst die PostgreSQL-Zugangsdaten zu SnapCenter hinzufügen.

.Schritte
. Wählen Sie im linken Navigationsbereich *Ressourcen* und dann SnapCenter Plug-in für PostgreSQL-Cluster aus der Liste aus.
. Wählen Sie auf der Seite Ressourcen den Ressourcentyp aus der Liste *Ansicht* aus.
. (Optional) Wählen Sie image:../media/filter_icon.gif["Filtersymbol"] aus und wählen Sie den Hostnamen aus.
+
Anschließend können Sie image:../media/filter_icon.gif["Filtersymbol"] das Filterfenster schließen.

. Wählen Sie den Cluster aus und wählen Sie dann *Anmeldeinformationen konfigurieren*.
. Wählen Sie die Anmeldeinformationen aus, die in SnapCenter für den PostgreSQL-Clusterbenutzer konfiguriert wurden. Die Anmeldeinformationen müssen einen gültigen Benutzernamen und ein gültiges Passwort mit ausreichenden Berechtigungen zum Durchführen von Datenschutzoperationen enthalten.
. Wählen Sie *OK*.


Die Clusterkonfiguration kann auf der Seite „Topologie“ geändert werden.
